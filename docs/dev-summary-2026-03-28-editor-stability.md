# 2026-03-28 节点编辑态稳定性修复总结

## 本次修复内容

- 修复了新建节点后约 1 秒自动退出编辑态的问题
- 保留了 autosave 回包后正在编辑的节点标题草稿与选区
- 节点标题内联编辑器改为自适应 `textarea`，支持随内容扩展与换行
- 根节点下新建分支时，初始落点改为优先左右均衡，和整理布局风格更接近

## 关键原因

问题根因不是新建节点本身，而是：

1. 新建节点后立即触发 autosave
2. autosave 成功后重新渲染节点层
3. 旧编辑器在重渲染时触发 `focusout`
4. 这次由重渲染引起的失焦被误判成“用户完成编辑”
5. 编辑态因此被清空，退回普通选中态

## 修复方式

- 在 `saveDocument` 前后捕获并恢复当前节点编辑草稿
- 对节点编辑器的 `focusout` 改为延后判断
- 如果发现同一个节点的新编辑器已经在重渲染后接管焦点，则跳过这次旧编辑器提交
