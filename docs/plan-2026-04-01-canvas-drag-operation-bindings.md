# 2026-04-01 画布操作绑定改为按住拖动计划

## 目标

- 将“操作绑定”从左中右键长按改为左中右键按住拖动，不再依赖长按延迟。
- 保持节点区域“手势绑定”仍然使用原有长按逻辑，不影响节点 AI 轮盘等动作。
- 修正命名和配置语义，避免画布操作继续以 `long press` 的方式表达。
- 完成补丁版本升级、重新打包并提交。

## 任务拆分

1. 配置与命名整理
   - 将画布绑定字段从 `canvas*LongPressAction` 调整为 `canvas*DragAction`
   - 保留旧配置兼容读取，避免本地用户设置丢失
2. 画布交互实现
   - 移除画布长按状态机和延迟触发
   - 在空白画布区域按下后，直接按绑定进入拖动画布或框选流程
   - 右键绑定到画布操作时抑制上下文菜单
3. 设置面板与文案
   - “操作绑定”使用“左键按住拖动 / 中键按住拖动 / 右键按住拖动”文案
   - 选项仍为“无操作 / 框选节点 / 拖动画布”
4. 交付验证
   - 构建前端
   - 执行 Go 测试
   - 升级补丁版本并重新导出桌面包

## 技术方案

- `frontend/src/types.ts`
  - 新增或调整画布拖动动作类型，交互设置改为 `canvasLeftDragAction` 等字段。
- `frontend/src/preferences.ts`
  - 默认值保持左键框选、中键拖动画布、右键无操作。
  - 读取时兼容旧的 `canvas*LongPressAction` 字段，并继续兼容旧版节点 `pan-canvas` 迁移。
- `frontend/src/app.ts`
  - 空白画布 `pointerdown` 时直接根据绑定启动 `startCanvasPan(...)` 或 `startMarqueeSelection(...)`。
  - 删除 `setTimeout` 驱动的画布长按状态机。
- `frontend/src/i18n.ts`
  - 为操作绑定增加“按住拖动”标签文案。
