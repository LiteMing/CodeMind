# 2026-03-29 布局吸附、折叠整理、历史快照与节点编辑开发计划

## 目标

围绕当前脑图编辑体验，完成以下 6 项调整：

- 自动创建节点时，同级节点上下尽量贴合
- 折叠/展开节点后可自动整理布局，且默认开启
- 节点拖拽增加 `x/y` 自动吸附，降低手动对齐成本
- 新建节点的自动避让只允许向下顺延，不再出现向上绕行
- 增加历史快照，支持手动保存和自动保存
- 节点编辑改为左侧固定、向右扩展，并恢复 `Shift+Enter` 手动换行

## 实施范围

### 1. 自动创建节点与避让逻辑

- 收紧新建子节点、同级节点和自由节点的默认纵向间距。
- 基于节点实际尺寸计算落点，不再简单按固定行高跳位。
- 保持“沿分支外侧展开”的方向策略，同时只做向下扫描避让。

### 2. 折叠后自动整理布局

- 在本地偏好里新增折叠后自动整理布局的交互开关，默认开启。
- 折叠/展开节点时，如果开关开启，则把折叠状态变更和自动布局作为同一次可撤销操作处理。

### 3. 拖拽吸附

- 为节点拖拽增加 `x/y` 吸附。
- 优先吸附父节点列、兄弟节点列和附近节点的横纵坐标。
- 只影响拖拽中的最终落点，不改变现有自由拖拽能力。

### 4. 历史快照

- 新增本地快照存储，不改动现有文档接口。
- 提供手动保存快照与恢复快照入口。
- 增加自动快照开关，默认开启，并控制频率与数量上限。

### 5. 节点编辑器

- 保持节点左边缘稳定，编辑时根据文本向右扩展。
- 恢复 `Shift+Enter` 插入换行，`Enter` 继续用于完成编辑。
- 让编辑态尺寸预览与只读态节点尺寸保持一致。

## 验证计划

1. `frontend` 执行 `npm run build`
2. 根目录执行 `go test ./...`
3. 执行 `wails build -nopackage`
4. 手工检查：
   - 左侧分支继续新建子节点时，新节点仍向左展开，且同级更紧凑
   - 折叠/展开节点后，默认会自动整理布局；关闭开关后恢复手动布局
   - 拖拽节点靠近同列/同行时会吸附，不再明显错位
   - 手动保存快照、自动保存快照、恢复快照都可用
   - 节点编辑时左边缘稳定，`Shift+Enter` 可换行
