# 2026-04-01 开发计划：父节点宽度避让子节点列

## 目标

- 修复子节点在自动创建后、以及自动整理布局后，可能贴住或压进被加宽父节点的问题。
- 让子节点列位置基于父节点实际左右边缘，而不是父节点中心点。

## 问题拆解

### 1. 新建子节点入口

- `nextChildPosition(...)` 最终通过 `resolveChildColumn(...)` 计算子节点列。
- 当前列起点使用 `parent.position.x +/- NODE_GAP_X`，忽略了父节点真实宽度。

### 2. 自动整理布局入口

- `autoLayoutHierarchy(...)` 的 `layoutBranch(...)` 按根节点固定列距递归排布。
- 当前深度列只按 `depth * NODE_GAP_X` 计算，未参考父节点实际宽度。

## 方案

- 新增统一的“相对子节点列边缘”计算逻辑：
  - 向右分支：使用父节点右边缘 + `NODE_GAP_X`
  - 向左分支：使用父节点左边缘 - `NODE_GAP_X`
- `resolveChildColumn(...)` 与 `layoutBranch(...)` 共用该逻辑，确保“新建时”和“自动整理后”行为一致。

## 验证

- `npm run build:web`
- `go test ./...`
- 手工检查：
  - 放大父节点宽度后再新建子节点，子节点应避开父节点边缘
  - 对同一分支执行自动整理，子节点不应回退到父节点上方
