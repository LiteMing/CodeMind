## 目标

- 修复区域框无法创建、无法稳定持久化的问题。
- 修复关系线无法左键选中、无法设置箭头方向、无法从中点建立分叉的问题。
- 保证区域框和关系扩展字段在前后端文档结构中可保存、可恢复。

## 任务拆分

1. 对齐文档模型
   - 前端 `types.ts`、`api.ts` 增加区域框与关系分叉字段规范化。
   - 后端 `internal/mindmap/document.go` 增加区域框、箭头、中点、分叉分支字段与基础校验。

2. 修复区域框交互
   - 修复右键创建后左键拖拽建立区域框的流程。
   - 修复区域框拖拽时节点联动与渲染坐标一致性。
   - 保持区域框位于底层，并继续支持 3D 视图染色外圈。

3. 修复关系线交互
   - 修复 SVG 关系线左键命中和选中高亮。
   - 支持无箭头 / 单向 / 双向箭头。
   - 在选中关系线后显示中点圆点，支持拖动调整。
   - 通过中点圆点长按拖拽到目标节点建立分叉连接。

4. 验证
   - 执行前端构建。
   - 如需补充，执行 Go 测试或至少保证 Go 编译通过。

## 技术方案

- 关系分叉采用“主关系 + 分支目标列表”建模，避免当前仅保存自由坐标点导致无法稳定连接节点。
- 中点圆点区分两种模式：
  - 普通拖拽：移动关系中点。
  - 长按后拖拽：从中点向目标节点建立分叉。
- 区域框节点归属采用节点包围盒与区域矩形相交判定，避免仅用节点中心点导致命中不稳定。
