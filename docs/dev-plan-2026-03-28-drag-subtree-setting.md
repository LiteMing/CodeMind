# 2026-03-28 父节点拖拽联动子节点设置开发计划

## 目标

在设置中新增一个交互项：

- 拖拽父节点时，自动一并拖拽其子节点
- 默认开启

## 实施范围

### 1. 偏好设置

- 在本地偏好中新增交互配置。
- 为旧配置提供兼容默认值，避免升级后字段缺失。

### 2. 设置面板

- 在设置面板中增加该选项。
- 修改后立即保存，无需重启。

### 3. 拖拽逻辑

- 在开始拖拽节点时，根据设置决定是否自动把当前节点的全部后代加入本次拖拽集合。
- 需要处理多选和父子重复选择，避免重复位移。
- 根节点仍保持不可拖拽。

## 验证计划

1. `frontend` 执行 `npm run build`
2. 手工检查：
   - 默认开启时，拖拽有子节点的父节点，整棵子树同步移动
   - 关闭后，仅移动被直接拖拽或被显式选中的节点
   - 多选父子节点时不出现重复位移

## 输出

- 代码改动
- `docs` 总结文档
