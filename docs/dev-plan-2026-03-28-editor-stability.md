# 2026-03-28 节点编辑态稳定性修复计划

## 背景

根据用户回归测试，`新建节点后编辑态容易丢失` 仍未完全修好。现象是新建节点进入全选文本编辑后，会在约 1 秒内退出为普通选中态。

## 初步结论

- 新建节点后会立即触发 `scheduleAutosave`
- autosave 完成后 `saveDocument` 会替换当前文档并重新渲染
- 旧的节点编辑器在重渲染时触发 `focusout`
- `focusout` 又把这次“由重渲染引发的失焦”误判成“用户完成编辑”，从而清掉编辑态

## 执行步骤

1. 修复节点编辑器在重渲染期间的失焦竞争，避免 autosave 回包把编辑态误关掉
2. 保持点击空白区、切换选择、点击工具按钮时仍然能够正常提交标题
3. 执行构建与测试验证
4. 重新打包桌面产物
