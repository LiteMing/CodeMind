# 2026-03-29 快照命名功能计划

## 目标

让本地历史快照支持命名，并在现有右侧面板内完成保存与展示，不引入额外弹窗流程。

## 实施步骤

1. 扩展本地快照存储结构，区分快照名称和脑图标题，并兼容已有旧快照数据。
2. 在右侧快照面板新增命名输入框，手动快照保存时优先使用用户输入名称。
3. 自动快照继续自动命名，但也写入标准化名称字段，保持列表显示一致。
4. 在脑图切换、返回主页、恢复快照等状态切换下重置命名输入，避免串值。

## 验证

1. `cd frontend && npm run build`
2. `go test ./...`
3. `npm run build:desktop`
4. 手动保存命名快照后，确认列表显示自定义名称并可恢复。
