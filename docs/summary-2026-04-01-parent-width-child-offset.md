# 2026-04-01 开发总结：父节点宽度避让子节点列

## 完成内容

- 修复了子节点列仍按父节点中心点计算的问题。
- 新建子节点时，列位置会基于父节点实际左右边缘留出间距。
- 自动整理布局时，递归分支也改为基于父节点实际宽度继续排布，避免整理后重新压回父节点。

## 关键实现

- `frontend/src/document.ts`
  - 为子节点列增加基于父节点真实宽度的相对边缘计算
  - `resolveChildColumn(...)` 改为使用父节点实际边缘而非中心点
  - `layoutBranch(...)` 改为按父节点边缘递归推导下一列

## 验证结果

- `npm run build:web`：通过
- `go test ./...`：通过

## 后续建议

- 若后续继续增强布局，可考虑把 `NODE_GAP_X` 从固定值升级为与父子节点宽度共同决定的动态间距。
