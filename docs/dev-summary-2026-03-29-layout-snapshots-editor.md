# 2026-03-29 布局吸附、折叠整理、历史快照与节点编辑总结

## 本次完成

### 1. 新建节点更紧凑

- 新建子节点、同级节点和自由节点不再按固定大步长落位。
- 改为根据已有节点实际高度计算下一行位置，并以更小步长仅向下顺延避让。
- 这样同级节点会更贴近，不再轻易留下过大空白。

### 2. 折叠后自动整理布局

- 新增本地交互设置 `interaction.autoLayoutOnCollapse`，默认开启。
- 折叠/展开节点后，如果开关开启，会立即执行层级整理布局。
- 折叠状态和整理布局被合并为同一次可撤销操作。

### 3. 拖拽吸附

- 新增本地交互设置 `interaction.dragSnap`，默认开启。
- 节点拖拽时会对附近的 `x/y` 目标进行弱吸附。
- 吸附目标包含父节点分支列、兄弟节点列，以及附近节点的横纵坐标。

### 4. 自动避让只向下

- 新建节点落位继续保持“沿分支外侧展开”。
- 避让扫描只向下推进，不再向上回绕。

### 5. 历史快照

- 新增本地快照存储模块，不改后端文档格式。
- 右侧面板增加“历史快照”区域，支持手动保存和恢复。
- 新增本地交互设置 `interaction.autoSnapshots`，默认开启。
- 自动快照在成功保存后按时间节流写入本地，避免过于频繁。

### 6. 节点编辑器

- 恢复 `Shift+Enter` 手动换行，普通 `Enter` 继续用于完成编辑。
- 编辑时会锁定当前左边缘，文本增长时向右扩展。
- 提交后会同步更新节点中心点，保持左边缘稳定，不再回弹到居中扩展。
- 节点宽高估算改成基于文本内容和多行情况，避免继续沿用旧的单行模型。
