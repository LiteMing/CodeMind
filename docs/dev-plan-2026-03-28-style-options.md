# 2026-03-28 风格功能开发计划

## 目标

为编辑器增加可持久化的风格设置，覆盖以下三项能力：

1. 在默认曲线连线之外，新增“方框直角连线”。
2. 增加“节点只在右侧延展排列”的布局模式，避免整理布局后根节点两侧同时分布。
3. 为顶部面板增加预设位置，先支持左侧、顶部居中、右侧三档。

## 实施范围

### 1. 偏好设置与持久化

- 扩展 `AppPreferences`，新增外观配置对象。
- 在 `preferences` 的默认值和加载逻辑中补齐兼容处理，避免旧本地配置报错或丢字段。
- 在设置面板中加入风格相关选项，修改后立即保存到本地。

### 2. 连线风格

- 保留现有曲线连线作为默认值。
- 为层级连线和关系连线新增直角路径生成逻辑。
- 调整关系连线标签定位，使其在直角模式下仍落在合适位置。

### 3. 右侧单边布局

- 扩展自动布局接口，使其支持“平衡双侧”和“仅向右延展”两种模式。
- 让根节点下新建子节点的位置也遵循当前布局模式，避免新增和整理布局表现不一致。
- 维持非根节点分支沿父节点方向延展，不改变现有分支层级规律。

### 4. 顶部面板位置

- 基于预设位置实现顶部面板对齐方式切换，不做自由拖拽。
- 保持现有浮动检查器与顶部面板的避让关系。
- 移动端继续退化为原有单列布局，不强行保留桌面端预设对齐。

## 验证计划

1. 前端构建：`npm run build`
2. Go 测试：`go test ./...`
3. Wails 打包：`wails build`
4. 手工检查：
   - 设置切换后刷新页面仍保留所选风格。
   - 直角连线在层级线、关系线场景都正常显示。
   - “整理布局”在右侧模式下不会把根节点子树分到左边。
   - 顶部面板切到居中后，检查器顶部避让仍正常。

## 输出

- 代码改动
- 打包产物
- `docs` 总结文档
