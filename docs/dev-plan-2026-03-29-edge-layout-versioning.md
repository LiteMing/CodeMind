# 2026-03-29 连线锚点、布局对齐与版本化打包开发计划

## 目标

完成以下 3 项调整：

- 节点根据文本扩展时，节点背后的层级连线不要跟着漂移
- 整理布局后，同级子节点需要进一步对齐
- 桌面打包改为 `x.y.z` 版本命名，当前版本定为 `1.0.0`，并确保产物带应用图标

## 实施范围

### 1. 连线锚点稳定

- 将层级连线从“节点中心到节点中心”改为“分支入口侧到分支入口侧”。
- 让节点宽度变化时，入线侧锚点保持稳定，避免出现节点变宽后背后连线一起横移。

### 2. 自动布局对齐

- 调整层级自动布局的横向计算，不再只对齐节点中心。
- 让右侧分支的同级节点按左边缘对齐，左侧分支按靠近父节点的一侧对齐。
- 新建子节点时继续沿用同一套列对齐逻辑。

### 3. 版本化打包

- 在 Wails 项目元数据里显式记录 `ProductVersion = 1.0.0`。
- 新增桌面打包脚本，统一产出版本化 exe 文件名。
- 打包前检查 `build/windows/icon.ico` 是否存在，并在构建后验证 exe 是否可提取关联图标。

## 验证计划

1. `cd frontend && npm run build`
2. `go test ./...`
3. 运行版本化桌面打包脚本
4. 手工检查：
   - 节点文本变宽后，背后的层级连线不再跟着横向漂移
   - 整理布局后，同级节点横向更整齐
   - 输出产物为带版本号的 exe，且图标可正常显示

## 输出

- 代码改动
- 版本化桌面产物
- `docs` 总结文档
