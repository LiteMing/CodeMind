# 2026-04-01 画布操作绑定与编辑延展计划

## 目标

- 让画布区域的拖动画布与框选支持按键长按绑定，默认中键长按拖动画布、左键长按框选。
- 将节点区域与画布区域的绑定配置拆开，避免“手势绑定”仅对节点生效却混入画布动作。
- 修正节点编辑态的自适应宽度预览，使节点左侧固定、仅向右延展。

## 任务拆分

1. 交互设置建模
   - 为画布区域新增独立绑定字段。
   - 保留旧版 `longPressAction` / 分键长按配置的兼容迁移。
2. 画布事件实现
   - 在画布空白区域识别左中右键长按动作。
   - 支持拖动画布与框选两类画布动作。
   - 更新设置面板，拆分“手势绑定”和“操作绑定”。
3. 编辑态宽度修复
   - 在节点编辑预览宽度变化时同步更新节点容器锚点。
   - 保持提交前后的节点左边缘一致。
4. 验证与交付
   - 运行前端构建与 Go 测试。
   - 如可打包则升级补丁版本并导出桌面构建。

## 技术方案

- 在 `frontend/src/types.ts` 中为交互设置增加画布绑定动作类型，节点动作继续沿用现有 `GestureAction`。
- 在 `frontend/src/preferences.ts` 中设置默认值为：
  - 画布左键长按：框选
  - 画布中键长按：拖动画布
  - 画布右键长按：无操作
- 在 `frontend/src/app.ts` 中拆分节点长按与画布长按状态机：
  - 节点区域继续负责节点动作与拖拽。
  - 画布区域新增长按触发框选 / 拖动画布，并兼容右键上下文菜单抑制。
- 编辑态宽度预览沿用已有 `activeEditorAnchorLeft`，在 `syncNodeEditorPreview` 中实时写入节点卡片定位。
