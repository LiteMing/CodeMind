# 2026-04-02 编辑态左侧锚点预览修复计划

## 目标

- 修复自动宽度节点在编辑模式下再次出现的“左右同时延长”问题。
- 保证节点编辑时左侧固定、右侧延展在持续输入和重渲染后都稳定成立。
- 修复编辑预览时连线仍按旧中心点计算导致的视觉偏差。

## 任务拆分

1. 编辑态预览建模
   - 增加当前编辑节点的预览几何状态
   - 保存预览宽高与左侧锚点
2. 节点与连线渲染修复
   - 节点渲染优先使用编辑态预览尺寸
   - 层级连线和关系连线优先使用编辑态预览中心点
3. 编辑状态切换
   - 打开编辑时初始化预览状态
   - 保存、恢复、退出编辑时同步清理或恢复预览状态
4. 交付验证
   - 前端构建
   - Go 测试
   - 升级补丁版本并重新打包桌面端

## 技术方案

- 在 `frontend/src/app.ts` 中增加编辑态预览状态，记录：
  - `nodeId`
  - `anchorLeft`
  - `width`
  - `height`
- `syncNodeEditorPreview(...)` 更新该状态，并即时重绘边层。
- `renderNodes()`、`renderEdges()` 改为读取预览几何，避免渲染时回退到节点旧中心。
