# 2026-03-29 连线与桌面图标修复计划

## 背景

用户反馈两项问题仍未修复：

1. 自动布局后的层级连线仍然杂乱，同级子节点没有形成稳定的统一分支竖线。
2. Windows 桌面包 `exe` 仍未正确显示应用 `ico` 图标。

## 目标

1. 修复同级节点布局与层级连线，使右侧/左侧分支都基于一致的边缘对齐规则渲染。
2. 修复或增强 Windows 打包图标链路，确保输出的 `exe` 确实携带项目图标。
3. 按现有版本规则继续输出 `CodeMind-1.0.0.exe`。

## 实施步骤

1. 统一前端布局计算与渲染计算使用的节点宽高估算规则，排查“布局对齐但渲染错位”的问题。
2. 调整层级连线锚点与自动布局的对齐列逻辑，确保同级节点的连接入口在同一条分支线上。
3. 强化桌面打包脚本验证：对比源 `ico` 与生成 `exe` 提取图标的实际像素结果，不再只做空值判断。
4. 如现有 `ico` 资源本身不满足 Windows Shell 显示要求，则补齐图标资源格式并重新打包。

## 验证

1. `cd frontend && npm run build`
2. `go test ./...`
3. `npm run build:desktop`
4. 验证 `build/bin/CodeMind-1.0.0.exe` 的图标提取结果与源 `ico` 一致。
